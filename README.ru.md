# antigravity-context-mcp

Два часа кодишь с агентом. Фича почти готова. Ты в потоке.

Контекст переполняется. Сессия падает.

Открываешь новый чат. Агент смотрит на тебя как в первый раз. *«Привет! Чем могу помочь?»*

Всё, что вы вместе построили — решения, архитектура, двадцать файлов — исчезло. Объясняешь заново. Опять.

**Этот MCP-сервер возвращает Antigravity память.**

---

## Как это работает

Сессия упала. Новый чат. Пишешь:

> *«продолжи»*

Агент сам вызывает `recall`, читает последнюю задачу — что было сделано, что в процессе, что планировалось — и продолжает с того места, где оборвалось.

Нужна более широкая картина?

> *«что мы делали на этой неделе?»*

Агент вызывает `recall_all` — последние несколько сессий плюс накопленные знания о проекте.

Никаких путей, аргументов, команд. Просто говоришь по-человечески.

## Установка

Открой чат Antigravity и отправь ссылку на этот репозиторий:

> *«установи этот MCP-сервер: https://github.com/eglebov133/antigravity-context-mcp»*

Агент сам склонирует, установит зависимости и пропишет конфигурацию. Перезапусти Antigravity — готово.

<details>
<summary>Ручная установка</summary>

```bash
git clone https://github.com/eglebov133/antigravity-context-mcp.git
cd antigravity-context-mcp
npm install
```

Добавить в `~/.gemini/antigravity/mcp_config.json`:

```json
{
  "mcpServers": {
    "context": {
      "command": "node",
      "args": ["/путь/до/antigravity-context-mcp/server.js"]
    }
  }
}
```

Перезапустить Antigravity.

</details>

## Что он читает

Сервер берёт то, что Antigravity уже хранит, но не использует между сессиями:

- Чек-листы задач — что сделано, что в процессе
- Итоги работы — описания завершённых задач
- Планы реализации — принятые решения

Всё локально. Никакой сети. Просто файлы на твоём диске, которые всегда там были.

При этом ответ сервера ограничен ~4000 символов — чтобы не перегрузить контекст нового чата той же информацией, от которой упал предыдущий.

## Помимо контекста

**Пароли между сессиями.** Знакомо — даёшь агенту пароль от хостинга, сессия умирает, новый агент снова спрашивает. Сервер хранит их в файле `.credentials` в проекте, автоматически исключённом из Git.

**Снимок контекста.** Агент может сохранить `AGENT_CONTEXT.md` прямо в проект — память, которая переживёт любое количество падений.

**Список проектов.** Показывает что Antigravity знает, чтобы переключаться без потери времени.

## Безопасность

- Работает локально через stdio — никаких открытых портов
- По умолчанию только чтение — пишет только по явному запросу
- Пароли остаются на твоей машине, исключены из Git
- Ноль телеметрии, ноль внешних запросов

- Node.js 18+
